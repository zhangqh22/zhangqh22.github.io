# 保存/命名

  1. 所有文件需要存放到公司网盘，方便以后查找使用。

  2. 文档最好带有设计师姓名，这样可以直接找到对应负责人

    • 文件夹（如果文件夹所有文件都是某个设计师做的，那文件夹就加设计师姓名）

    • 文件（如果只做了某个文件，那就在文件后面加设计师姓名）

    另外，对于直接打包到前端的UI文件，最好都带个"_UI"后缀，以防和产品的UE文档命名冲突，而且比较直观，示例如下：

    珠海大屏.rar -> 珠海大屏_张三_UI.rar

# 图标/命名

  1. 字体图标需要上传到阿里图标库，减少重做的可能，方便选择下载使用。

  2. 项目名/图标库名以 fy_ 开头，下划线间隔，如：fy_weather_small

├── fy_weather_small
├── fy_weather_large
└── fy_common

# 字体

  1. 统一使用微软雅黑，为达到特殊效果使用特殊字体的除外。
  2. 字体大小最好不要有奇数的
body {
    font-family: "Microsoft YaHei";
    font-size: 14px;
}
